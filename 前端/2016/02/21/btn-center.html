按钮数量不定居中显示

2016-02-21

如果让你写一段代码，来实现三个按钮居中显示，你是否会这样实现：

<div class="box">
  <a href="#">增加</a>
  <a href="#">删除</a>
  <a href="#">修改</a>
</div>

按钮的样式为：

.box {
  width: 300px;
  margin: 0 auto;
  font-size: 0;
}

.box a {
  display: inline-block;
  width: 98px;
  height: 30px;
  color: #0075eb;
  font-size: 14px;
  line-height: 30px;
  text-align: center;
  border: 1px solid #0075eb;
  border-radius: 2px;
  background: #fff;
  text-decoration: none;
}

当然如果按钮之间还需要10像素的间距，可以这么做：

<div class="box box-2">
  <a href="#">增加</a>
  <a class="ml-10" href="#">删除</a>
  <a class="ml-10" href="#">修改</a>
</div>

按钮样式：

.box-2 {
  width: 320px;
}

.box .ml-10 {
  margin-left: 10px;
}

好了，我们已经觉得代码完美无缺了，那我就来幸灾乐祸地提出几个问题吧。

  • 1、如果我想把按钮都改成100像素的宽，保持居中，怎么做？
  • 2、如果我想增加或减少一个按钮，保持居中，怎么做？
  • 3、如果我想增加或减少多个按钮，保持居中，怎么做？


不问了，很明显，上面的方案不是一个很好的方案。如果要改动一下按钮的样式，特别是宽，那父元素的样式也要跟着改。那么我们就来看看下面这种方案吧，对比一下看看下面的方案有什么优势：


首先，html的格局不变，而按钮的样式稍作修改，给所有的按钮都设置一个左间距。

.box a {
  // ...

  margin-left: 10px;
}

然后是父元素的样式，这里父元素不需要宽度和居中对齐，并且给父元素设置一个负的左间距，用来抵消第一个按钮的左间距，然后设置一个文本居中。

.box {
  margin-left: -10px;
  text-align: center;
  font-size: 0;
}

来看下效果吧：

增加 删除


增加 删除 修改


给父元素设置了一个文本居中就可以让inline-block的元素居中了，有点神奇！这种方案相比于上一个方案有几个优势：

  • 1、按钮的宽度可以随意修改；
  • 2、按钮的数量可以随意增加或减少；
  • 3、低版本浏览器也可以兼容。


彩蛋！

你是否还在纠结为什么inline-block的元素之间会有间隙，而且去不掉吗！上面已经有了答案，快给父元素设置一个0的字体吧！更多解决方案请看张鑫旭博客。
